1 The Problem

Project boards fill up fast. Tasks get added without priorities, descriptions are vague, and team members spend time in planning meetings deciding what to work on next — a conversation that could largely be automated. Manually triaging a backlog is slow and inconsistent. Different people prioritize differently, and nothing gets labeled until someone makes time for it.

The goal was to build a system that automatically evaluates every new task the moment it's created and applies consistent, structured priority labels — so the board is always sorted and ready to work from.

2 How It Works

The workflow triggers whenever a new task is added to Asana and runs it through an AI evaluation pipeline before writing the results back.

Workflow Flow

Asana Trigger (New Task) → Get All Project Tasks → Prepare Task Context → AI Task Prioritizer (GPT-4) → Structured Output Parser → Extract Prioritized Tasks → Update Task Priority in Asana → Add Tags

The agent fetches all current project tasks so it can evaluate the new task in context — it knows what already exists, what's urgent, and where the new task sits relative to the rest. GPT-4 returns a structured JSON output with priority score, effort estimate, and recommended tags. These are extracted and written directly back to Asana via API — updating the task's priority and adding appropriate labels automatically.
